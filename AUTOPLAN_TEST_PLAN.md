## Affected Commands

- `autoxmimsim run-real-bronze-demo tests\fixtures\CuSnBronze.xmsi --output reports\real-bronze-demo`
  verifies real XMI-MSIM target/candidate orchestration using a tiny fixed candidate set.

## Key Interactions To Verify

- Running the command creates `target/` and `candidate-000/` style artifact folders.
- Each candidate folder contains the rendered `.xmsi`, produced `.xmso`, and converted `.csv`.
- CLI output prints best score, best parameters, report path, and artifact root.
- The report or JSON summary lists every evaluated candidate and its artifact paths.

## Edge Cases

- `xmimsim-cli.exe` returns non-zero.
- `xmso2csv.exe` returns non-zero.
- Rendered XMSI is missing a required output path.
- CSV export is empty or malformed.
- Candidate spectra use an energy grid different from the target.
- A candidate fails while previous artifacts already exist.

## Critical Paths

- Target render -> XMI-MSIM -> CSV -> `Spectrum`.
- Candidate render -> XMI-MSIM -> CSV -> `Spectrum` -> score.
- Best candidate selection from 2-3 candidate scores.
- Error messages preserve command, stdout, stderr, and artifact path context.
